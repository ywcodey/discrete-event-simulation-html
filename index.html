Discrete Event Simulation

This page is used to show the code and result of a queue simulation project. By implementing discrete event simulation, we are going to model and discuss the event of packets arrival and departure in a computer network.

The project includes
  • generation of random variables for a given distribution
  • building an event scheduler
  • discussion for the behavior of a single buffer queue with different parameters

The M/M/1 and M/M/1/K model would be built in this project.

This page is written in HTML, CSS, Bootstrap.

The simulation code is written in Python. The code can be found in here.

M/M/1 Simulator

  • 1. Create a function to generate exponential random variables, and create an linked list object as event scheduler (ES) list.
  • 2. Generate packet arrival events, with parameter arrival rate, and append to ES, until reaching the time length.
  • 3. Generate observe event, with parameter observe rate, and insert to ES, until reaching the time length.
  • 4. Initialize variables to zero, for example, queue length, number of arrivals, number of observe, number of departures.
  • 5. Dequeue one event at a time from ES and call the corresponding event procedure.
  • 6. Obtain the output result for further discussion and analysis.

M/M/1/K Simulator

  • M/M/1/K, where K denote the size of the buffer in number of packets.
  • Since the buffer is finite, when buffer is full, arriving packets will be dropped.
  • So, two variables are introduced to the simulator, buffer_length and num_of_drop.

M/M/1 Simulator

  • Assume L=12000bits, C=1Mbits/second, where L is average length of a packet in bits, and C is the transmission rate of the output link in bits per second.
  • The service rate is 1000000/12000=83.3333
  • rho = arrival rate/service rate = 75 / 83.3333 = 0.9
  • For the first plot, here is E[N], the average number of packets in the system as a function of rho.
  • Given rho and arrival rate, service rate can be calculated.
  • So, with a fix arrival rate and changing service rate, the plot can be obtained.
  • E[N] can be obtained by queue length recorded by observe event divided by number of observe event.
  • When rho increases, the E[N] increases.
  • When rho is 0.9, the E[N] is 9, which mean average number of packets in the system is 9.

  • For the second plot, it is the proportion of time the system is idle as a function of rho.
  • The proportion of idle can be obtained by number of idle recorded by observe event divided by number of observe event.
  • When rho increases, proportion of idle decreases.
  • When rho is 0.9, proportion of idle is close to 0.1.

M/M/1/K Simulator

  • For the first plot, here is E[N], the average number of packets in the system as a function of rho for K=5, 10, 40.
  • It can be found that since K=40 has buffer length 40 which can accommodate more packet rather than drop the packets. So, the green line increases more rapid than the other two lines.

  • For the second plot, when including infinite buffer length to K=5, 10, 40, it can be found that since infinite buffer length can store infinite packets, as rho increases, the E[N] also increase.

  • For the third plot, this is the probability of packet loss for rho ranging from 0.4 to 10. The probability of packet loss is calculated by number of drops divided by number of packets.
  • It can be found that after rho greater than 2, the different in buffer length does not result in different of probability of packet loss.
  • While for rho less than 2, for example rho is 0.9, different in buffer length to probability of packet loss do matter.
  • Longer buffer length can result in lower probability of packet loss, while shorter buffer length has higher probability of packet loss.
